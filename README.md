# Wasiy

Wasiy is an operations-first SaaS product for residential properties such as condos, multifamily buildings, and residential communities. It helps property teams manage residents, units, visitors, amenity reservations, vehicles, announcements, activity logs, and basic operational reporting across one or more locations.

The v1 product is Spanish-first and designed for three main surfaces:

- Admin and Location Manager dashboard
- Front Desk / Security interface
- Resident Portal

## Repository Structure

```txt
apps/
  api/        Laravel REST API
  web/        Vite React SPA
  marketing/  Astro marketing site
docs/
  adr/        Architecture decision records
```

Current package names:

- `wasiy/api`
- `@wasiy/web`
- `@wasiy/marketing`

## Stack

- Backend: Laravel 13, PHP 8.3, PostgreSQL, Redis, Laravel Sanctum, queues, Pest
- Web app: Vite, React, TypeScript, TanStack Router, TanStack Query, Mantine, Tailwind CSS, i18next
- Marketing: Astro
- Local services: Laravel Sail with PostgreSQL, Redis, MinIO, and Mailpit

## First-Time Setup

Install frontend workspace dependencies from the repository root:

```sh
pnpm install
```

Install API dependencies:

```sh
cd apps/api
composer install
cp .env.example .env
```

If using Sail, adjust `apps/api/.env` for container networking:

```env
APP_NAME=Wasiy
APP_URL=http://localhost

DB_CONNECTION=pgsql
DB_HOST=pgsql
DB_PORT=5432
DB_DATABASE=api
DB_USERNAME=root
DB_PASSWORD=secret

REDIS_HOST=redis
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
```

Start the API services:

```sh
./vendor/bin/sail up -d
./vendor/bin/sail artisan key:generate
./vendor/bin/sail artisan migrate
```

Useful local service URLs:

- API: `http://localhost`
- Vite through Sail, if used from the API app: `http://localhost:5173`
- Mailpit: `http://localhost:8025`
- MinIO console: `http://localhost:8900`

If port `80` is already in use, set `APP_PORT` in `apps/api/.env`, for example:

```env
APP_PORT=8000
```

Then point the web app at that API URL with `apps/web/.env.local`:

```env
VITE_API_BASE_URL=http://localhost:8000
```

## Running Locally

Run the API with Sail:

```sh
cd apps/api
./vendor/bin/sail up -d
```

Run the React web app from the repository root:

```sh
pnpm --filter @wasiy/web dev
```

Run the marketing site:

```sh
pnpm --filter @wasiy/marketing dev
```

Common local URLs:

- Web app: `http://localhost:5173`
- Marketing site: `http://localhost:4321`
- API: `http://localhost` or the configured `APP_PORT`

## Routing

The web app uses TanStack Router file-based routing.

Important files:

```txt
apps/web/src/app/router.tsx
apps/web/src/routeTree.gen.ts
apps/web/src/routes/__root.tsx
apps/web/src/routes/admin/route.tsx
apps/web/src/routes/admin/index.tsx
apps/web/src/routes/front-desk/route.tsx
apps/web/src/routes/portal/route.tsx
```

`routeTree.gen.ts` is generated by the TanStack Router Vite plugin. Do not edit it by hand.

## Layouts

The web app has separate product-area layouts:

```txt
apps/web/src/components/layout/
  admin/
  front-desk/
  portal/
  shared/
```

Use the area-specific layout folders for route shell composition, and keep reusable chrome primitives in `shared`.
